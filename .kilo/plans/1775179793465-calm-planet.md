## QA Monitoring Protocol

### 1. Continuous Monitoring Commands

```bash
# Check for new commits
git log --oneline -5 --all

# Check current build status
npm run build

# Run all checks
npm run lint && npm run test -- --run
```

### 2. Issue Detection Triggers

Alert when:

- Build fails (`npm run build` returns non-zero)
- Lint errors
- Test failures
- New commits merged to main that may introduce issues

### 3. Branch Tracking

Active feature branches to monitor:

- `feat/step-4-dogs` (current working branch)
- `main` (integration branch)
